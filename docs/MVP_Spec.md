# English Quote Hangman Game (MVP Spec)

------------------------------------------------------------------------

## 1. 목적 (Goal)

영어 명언을 행맨(Hangman) 방식으로 맞추며 철자 패턴과 어휘 감각을
학습하는 게임을 구현한다.\
동일 알파벳은 동일한 숫자/기호 힌트를 제공하여 초보자도 패턴 기반 추론이
가능하도록 설계한다.

MVP 범위는 **1인 플레이 / 로컬 데이터 / 단일 브라우저 실행**으로
제한한다.

------------------------------------------------------------------------

## 2. MVP 범위 (In Scope)

-   레벨 기반 명언 선택
-   빈칸 기반 문장 표시
-   동일 알파벳 → 동일 숫자/기호 힌트 매핑
-   드래그 앤 드롭 기반 알파벳 배치
-   카드 상태 변화(활성/회색 처리)
-   승패 판정
-   남은 시도 횟수 표시

### Out of Scope (MVP 제외)

-   사용자 로그인
-   점수 저장/DB 연동
-   음성/발음 기능
-   의미/해설 자동 제공
-   모바일 최적화
-   서버 통신 기능

------------------------------------------------------------------------

## 3. 게임 규칙 (Game Rules)

### 3.1 기본 규칙

-   한 라운드당 명언 1개를 사용한다.
-   알파벳(a--z)은 모두 빈칸(\_)으로 시작한다.
-   공백 및 구두점(. , ' ? ! -)은 그대로 표시한다.
-   사용자는 화면 하단의 **알파벳 카드**를 드래그하여 원하는 빈칸에
    배치한다.
-   대소문자 구분은 하지 않는다 (내부적으로 소문자 처리).

------------------------------------------------------------------------

### 3.2 암호 힌트 규칙 (Cipher Hint System)

-   해당 문장에 등장하는 **고유 알파벳 집합**을 추출한다.
-   각 고유 알파벳에 숫자(1--9) 또는 기호 집합 중 하나를 무작위
    매핑한다.
-   동일 알파벳은 동일 숫자/기호를 가진다.
-   매 라운드마다 매핑은 새로 생성된다 (라운드 고정 방식).

#### 예시

문장:\
`free`

매핑 예시:\
f = 5\
r = 2\
e = 3

표시:

------------------------------------------------------------------------

5 2 3 3

------------------------------------------------------------------------

## 4. 난이도 설계 (Level System)

### Level 1

-   문장 길이 ≤ 60자
-   무료 공개 글자 1개 제공
-   시도 횟수: 8회

### Level 2

-   문장 길이 60\~100자
-   무료 공개 없음
-   시도 횟수: 7회

### Level 3

-   문장 길이 ≥ 100자
-   시도 횟수: 6회

------------------------------------------------------------------------

## 5. 상태 정의 (Game State Model)

### 상태 값

-   INIT
-   PLAYING
-   WIN
-   LOSE

### 상태 전이

-   INIT → PLAYING\
-   PLAYING → WIN (모든 빈칸이 정확히 채워진 경우)\
-   PLAYING → LOSE (남은 시도 0)

------------------------------------------------------------------------

## 6. 입력 정책 (Drag & Drop Policy)

### 6.1 알파벳 카드 구성

-   화면 하단에 **해당 문장에 포함된 고유 알파벳 카드만 표시**
-   불필요한 알파벳은 표시하지 않음

### 6.2 드래그 앤 드롭 규칙

-   사용자는 알파벳 카드를 개별 빈칸에 직접 배치한다.
-   각 빈칸은 독립적으로 채워진다.
-   동일 알파벳이 여러 위치에 등장하더라도 자동 채워지지 않는다.
-   사용자가 각각 배치해야 한다.

### 6.3 카드 상태 변화 규칙

-   특정 알파벳이 문장 내 모든 위치에 정확히 배치되면:
    -   해당 알파벳 카드는 **회색(비활성 상태)** 으로 변한다.
-   아직 일부 위치가 비어 있거나 오배치된 경우:
    -   카드는 활성 상태 유지

### 6.4 오배치 정책

-   잘못된 알파벳을 빈칸에 놓을 경우:
    -   해당 빈칸은 빨간색 표시
    -   시도 횟수 1 차감
    -   카드가 원래 위치로 복귀

------------------------------------------------------------------------

## 7. 승패 조건

### WIN 조건

-   모든 빈칸이 정확한 알파벳으로 채워진 경우

### LOSE 조건

-   남은 시도 횟수가 0 이하가 된 경우

------------------------------------------------------------------------

## 9. 수용 기준 (Acceptance Criteria)

AC1. 문장 내 동일 알파벳은 동일 암호를 가진다.\
AC2. 구두점과 공백은 시작 시 표시된다.\
AC3. 알파벳 카드는 해당 문장에 포함된 고유 알파벳만 표시된다.\
AC4. 동일 알파벳이 여러 위치에 있더라도 자동 공개되지 않는다.\
AC5. 특정 알파벳의 모든 위치가 정확히 배치되면 해당 카드가 회색으로
변한다.\
AC6. 오배치 시 시도 횟수가 1 감소한다.\
AC7. 남은 시도 0이면 즉시 LOSE 상태가 된다.\
AC8. 모든 빈칸이 정확히 채워지면 WIN 상태가 된다.\
AC9. Level 1에서는 무료 공개 글자가 자동 적용된다.
